タイトルバーは非表示（信号機ボタンのみコンテンツ上に残る）。常駐 chrome は最上段の `StatusRow` 1 つ（タブ 1 枚でも常時表示）で 2 段からなる。上段は標準タイトルバー高に一致させ、信号機を帯の縦中央に置いて上下余白を対称化する。罫線は持たず、下段の地の濃度差が境界を成す。最背面には accent と working のラジアルグローが敷かれ、上段は透明でこれが透ける。背景透過時（非フルスクリーン）は StatusRow 全体に端末面と同濃度の veil を敷き、その上に既存のタブ行の地を重ねる——ステータス行の背景もデスクトップまで透ける（→ [layout](layout.md)・[config](config.md)）。不透明時は上段透明・下段はタブ行の地。chrome の全ユーザー向け文言は現在言語に追従する（[localization](localization.md)）。

**上段（TopBar）**: 左から信号機ぶんの余白・workspace 名・アクティブペインの実効 cwd（`currentPwd ?? initialCwd`・`~` 短縮・先頭側省略）＝現在地の粗→細、右端に全 workspace 横断のエージェント状態件数ストリップ（後述）、中央は空（窓ドラッグ面）。幅不足時は cwd から縮む（workspace 名・ストリップは保護）。テキストの縦位置は信号機 close ボタン中央へ寄せる。環境変数 `ORBE_STATE_DIR` 設定時のみ workspace 名の右隣に build-id（ビルド時の git 短縮 SHA）を dim 表示する——検証用隔離インスタンスの鮮度表示で、未設定の常用 Orbe では非表示（[persistence](persistence.md)）。

**下段（タブ行）**: 全幅のセグメント形タブ行。各タブはセグメント（タイトル末尾省略・クリックで切替）、末尾に `+` で新規。選択タブは前景色反転、非選択タブは淡いブロック。タブ背景を状態色で塗らない。タブはドラッグ&ドロップで左右に並び替えできる（同一 workspace 内のみ・実データのタブ配列そのものを入れ替え・順序は永続）。掴みは小さな移動量の閾値でクリック切替と区別し、掴んだタブを指に追従させ挿入先の隙間に accent 色の縦キャレットを出す。指を離した瞬間に 1 回だけ配列を並べ替える（commit-on-drop・ドラッグ中はデータを変異させない）。

**タブタイトルの解決**: ① 明示タイトル（`Cmd+R` で付与・tab 単位・sticky・空入力確定で解除）→ ② フォーカス中ペインのアプリ報告タイトル（OSC 0/2。ただし `currentPwd` と同値のときは採用しない＝libghostty が明示タイトル未受信の間に立てる pwd フォールバックを除外）→ ③ cwd 派生（所属 workspace の `rootPath` 基準で整形：圧縮アンカーを root の親に置く＝root 以下なら root 名を先頭要素とする相対・root 外は `~` 基準。root ちょうどは basename。いずれも fish `prompt_pwd` 流に末尾要素以外を頭 1 字へ圧縮・深さ上限なし）の優先順位で決まる。`Cmd+R` はフォーカス中タブをタブ行内でその場編集する（中央パレットは出さない）——現在の表示名をプリフィルし全選択して開き（打てば置換）、空確定で解除・Esc で取消・他所クリックで取消。編集中は window レベルの chrome コマンドを不活性化する。シェル統合の自動タイトル送出は既定 conf で止める（[config](config.md)）。

タイトルの描画は基底フォント（既定は等幅システム・設定 `tab-title-font-family` で差し替え可・未設定/解決不能名は既定へ退避・[settings-palette](settings-palette.md)）に、Nerd アイコン・点字・絵文字の各領域を run 単位で同梱フォント（絵文字は設定 `emoji-font` の実効値。apple 選択時は割り当てず OS フォールバック）へ明示割り当てる——ghostty 側と違い chrome は自前で解決するため。text 既定の emoji 可能記号（Claude Code の ✳ 等）は VS16 を足してカラー表示へ昇格する（タブ自然幅の計測も同じ変換・同じ基底フォントで測る。フォント未解決のバンドル無し起動では割り当てなし＝素の chrome 描画）。この run 割り当てはタブ以外の chrome テキスト——TopBar の workspace 名/cwd・パレット行・EditorPane のファイル名/パス・FileViewer の本文行——にも適用される（本文行は絵文字等を含まない文字列を走査の早期打ち切りで素の Text のまま描く）。

**タブのエージェント状態インジケータ**: 各タブはタイトルの前に、タブ内全ペインの `agentState` を `waiting > working > done` の優先順位で 1 つに畳んだ状態を状態グリフ（working=円弧スピナー・waiting=吹き出し浮遊・done=チェック・静止）で出す。Reduce Motion 環境ではアニメーションを止め静的な形で残す。`idle`・無し（nil）は出さない。状態の詳細（種別グリフ＋件数）は上段右端の横断ストリップが持つ。各状態のグリフは既定でこの自前字形だが、設定パレットの「エージェントアイコン」で状態ごとに SF Symbol へ差し替えられる（[settings-palette](settings-palette.md)）。差し替えても状態のモーション・Reduce Motion 時の静止・選択タブの対テーマ色は維持する。装飾用途の固定グリフ（EditorPane のコミット✓・dispatch の作成中スピナー等）は差し替え対象外。

**タブ overflow（比例縮小 ＋ 水平スクロール）**: タブ群は水平スクロール領域（スクローラー非表示）に載る。各タブの自然幅は上限で cap され、合計が利用可能幅に収まる時はそのまま。収まらない時は CSS flex shrink と同じく全タブを自然幅に比例して縮め（短いタブも縮む）、最小幅の床に達したタブは凍結して残りへ再配分する。全タブが床でも収まらない時だけ水平スクロールする。タブ切替時はアクティブタブを可視位置へ自動スクロールする。描画クリップでタブを隠さない（到達不能なタブを作らない）。

**done のフォーカス消費**: 見ているタブ（アクティブ表示タブ かつ ウィンドウがキー＝前面）には `done` を残さない。開いて見ている最中に `done` が届いた時・別タブから `done` のタブへ切り替えた時・前面復帰した時のいずれでも、そのタブ内全ペインの `done` を `idle`（休止）へ遷移させる。idle はタブにインジケータを持たないため集約 `done` バッジは消えるが、横断ストリップには休止中エージェントとして残る。背面（非キー）で届いた `done` はバッジが残り、前面復帰・切替で消える。`waiting`・`working` は消費せず残す。コマンド名・セッション ID は保持する（resume を壊さない）。

**全 workspace 横断エージェント件数（ストリップ）**: 全 workspace・全タブ・全ペインを走査し、`agentState` を `working / waiting / done / idle` ごとに件数集計する（ペイン単位。idle は数えるが `error`・nil は数えない）。この状態順で、状態色グリフ（CLI 非依存・タブと同じく設定で SF Symbol へ差し替え可 → [settings-palette](settings-palette.md)）＋件数を出す。項目は区切り線を持たず間隔で分ける。休止（idle）項目は減光する。ラベル文字は持たない。件数 0 の種別・全体 0 は出さない。上段右端に裸のグリフ列（囲みなし・非インタラクティブ）で固定表示する。同じ集計を WorkspacePalette の起動済み workspace 行にも固定 4 カラムの状態カウント表として出す（未起動行は休眠 zzz を休止カラムに表示 → [workspace](workspace.md)）。

行の余白ドラッグでウィンドウ移動、余白ダブルクリックでシステム設定準拠の zoom/minimize（[layout](layout.md) に機構）。更新はイベント駆動: ペインのタイトル変更・フォーカス変更・タブ/workspace 切替・cwd 到着・いずれかのペインのエージェント状態変化。重い StatusRow snapshot（全 workspace×全タブ×全ペイン走査）は runloop tick 単位に coalesce され、同一 tick 内の複数要求は tick 末尾の 1 回へ畳まれる（trailing-edge・中間状態は落ちるが最終状態は必ず反映＝取りこぼしゼロ）。`window.title`（アクティブ workspace 名・Mission Control 等の外部観測用・画面には出ない）は O(1) なので coalesce せず即時反映する。
