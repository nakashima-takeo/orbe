host 所有。ドメイン状態（workspace 配列とアクティブ index）の実体は Foundation 純粋型 `SessionStore` が所有し、配列 CRUD・active index 補正・MRU 退避先選定はその純メソッド経由で行う。`WindowController` は store を持つ薄いコーディネータで、ビュー mount/reparent・focus・chrome 投影・永続・制御チャネルを担う（タブ閉鎖・選択のような危険な操作は store が「決定（outcome）」を返し、controller がビュー副作用を実行する）。

起動時に既定 workspace 1 つ（または永続から復元 → [persistence](persistence.md)）。workspace は root path を持つ。

## 保持・切替

- **0 タブ（休眠）でもエントリを保持する**。0 タブ workspace のアクティブ化（切替・復元・削除後の MRU 繰上げ）は空状態を表示し、シェルは自動起動しない。シェルの起こしは新規作成の明示動作にのみ残す。タブを 1 枚も持たない workspace を配列から削除することはない。
- 非アクティブ workspace の `TerminalController` は保持され surface は生存する（同一セッション内 keep-alive）。切替は前 workspace のビューを外して切替先を載せるだけで、keep-alive 済みなら再生成ゼロ。背景 workspace は載せず休眠する。
- アクティブ workspace は**全タブ**の分割ツリーをウィンドウ階層に載せ、アクティブタブのみ可視・他タブは hidden にする——非アクティブタブもフォーカスを待たず surface が起動して復帰でき、タブ切替が可視/非可視のトグルだけで済む（surface を再生成しない）。
- 隠れタブ・背景 workspace・occluded ウィンドウの surface は**描画のみ**停止する（端末状態・pty は前進。表示復帰時に 1 フレーム描画。可視性同期の契約は [terminal-core](terminal-core.md)）。
- アクティブ化では可視タブを即時 mount し、未 mount の隠れタブは後続 runloop tick で 1 枚ずつ分割 mount する（1 turn で N 個の surface を同期生成しないため）。隠れタブも最終的に必ず mount され surface が起動する不変は保つ（resume 起動も走る）。分割 mount の途中で別 workspace へ切り替えたら進行中バッチは破棄し（孤児 mount を防ぐ）、次のアクティブ化で再 mount する（冪等）。
- 新規 workspace の root path は、クイック作成（切替パレットで一致なし名＋Enter）では active ペインの cwd 由来（不明時はホーム）、専用作成フォーム経由では入力パス。
- アクティブ workspace の最後のタブを閉じても、その workspace は 0 タブの空状態でアクティブに残る（単一・複数 workspace 問わず・ウィンドウは閉じない）。背景 workspace の最後のタブが閉じても 0 タブのまま残す。
- パレットの詳細メニューからの削除は、アクティブ workspace なら最近使った他 workspace（MRU）を次のアクティブにし、背景 workspace なら現アクティブは不変（workspace が 1 つだけなら削除不可）。

## WorkspacePalette（Cmd+Shift+S）

上端寄せ・水平中央のオーバーレイ（背後は暗幕＋ぼかし）。カードはガラスパネルで、ヘッダ行（絞り込み入力）／行リスト／フッターヒントからなる。入力欄と breadcrumb がともに無いパレットではヘッダ行ごと描かず、行ゼロなら行リストごと描かない（改名/ディレクトリ等の入力欄だけのプロンプトで空帯を作らないため）——いずれも共有 `PaletteCard` の挙動。

- **一覧**: 入力で workspace 名をインクリメンタル絞り込み（開くと選択カーソルがアクティブ workspace 行に載る）、↑↓ で選択、Enter で切替（一致なし入力＋Enter で新規作成）、workspace 行で → で同じカードが詳細メニュー（改名/ディレクトリ/削除・workspace が 1 つなら削除無し）へ潜る、Esc で閉じる。
- **詳細メニュー**: ↑↓ 選択、Enter 実行、←/Esc で一覧へ戻る。改名・ディレクトリへ潜ると入力欄へ focus が当たり（クリック不要でそのまま編集できる）、現値をプリフィルし Enter で確定（空確定は無視・編集中の Esc は詳細へ戻る）。ディレクトリ確定は root path を更新し、新規作成時のシェル起動 cwd とタブタイトルの相対パス基点に使う。
- 削除・改名・ディレクトリ確定などパレット表示中の変異でも、再読込後に focus を次 tick で一覧の入力欄へ再確定するため、直後の ↑↓/Enter が効く。削除後のハイライトはアクティブ workspace の行に合わせる（最上段固定ではない）。
- **焦点の常設**: `PaletteCard` は単一の focus 状態（`.field`／`.card`）で焦点を一意に保ち、**カード器と入力欄の両方を常に mount する**（入力欄の可視性は場所を取るかだけを決める）。宛先が常設なので、詳細メニュー↔一覧・サブパレット↔root の往復でも焦点が確実に移り、戻った直後の ↵・→・打鍵での絞り込みがそのまま効く。一覧/改名は `.field`、詳細メニューは `.card`。常設カード器は ←→↑↓↵Esc を捕捉するが、`.field` のときは ←→↵ を入力欄へ委ねる。
- ↑↓ 単独は 1 行移動、⌘↑↓ は選択を有効な先頭/末尾行へジャンプ（無効行は選ばず、有効行ゼロ/空は no-op）。この挙動は共有 `PaletteCard` のもので、設定パレット（[settings-palette](settings-palette.md)）・エージェント起動パレット（[agent-launch](agent-launch.md)）にも同じく及ぶ。行リストは高さ上限で頭打ちし、超過分は内部スクロール（選択行に追従・上限未満は中身をハグして余白を作らない）。
- **行の着色は選択 1 つだけで、ホバー専用の地色を持たない**——マウスホバーで選択がその行へ追従し（着色行は常に 1 つ・無効行へは追従しない）、行タップは決定（↵ と同じ経路）。ホバー追従は入力モダリティ（keyboard / pointer）が門で、実ポインタ移動で pointer へ移り、キー移動・絞り込み・ドリルイン等モデルが選択を置く操作では keyboard へ戻る。keyboard の間はホバーが選択を動かさないため、**キー操作でリストがスクロールして行がカーソル下へ来ても選択を奪われない**。ホバーは選択を動かすだけで決定は走らない。

### workspace 行

並びは `[名前] [インラインチップ群] [spacer] [パス]`。名前は統一色（状態や選択で色を変えない）、パスは行右端に寄せた root path（`~` 短縮・tail 省略・muted）。

チップ = 状態アイコン（Glass グリフ・状態色）＋（件数 ≠ 1 のとき `×N`）。件数 1 はアイコンのみ。存在する状態だけを正準順 `working → waiting → done → idle` で並べ、0 件の状態は出さない（囲み・地色は持たず、固定カラムの空欄も無い）。集計母数は [chrome](chrome.md) のストリップと同じ。パレットを開いた時点のスナップショットだが、表示中もタブ消滅・0 タブ化・削除・改名・ディレクトリ変更では実状態へ再読込する（agent 状態変化だけでは更新しない）。

**未起動（休眠）行では実 agentState が 0 件のため、代わりに永続している agent（保存ツリーの葉）の総数を idle 色チップ 1 つに一本化して出す**（0 件は出さない・未起動の見分けは行全体の減光が担う）。この件数は resume 未対応で復元時に素シェル化した agent も含む（復元時に葉の数を確定して数えるため取りこぼさない）。休眠 workspace は活性化前にミューテートされないので復元時に読んだ保存内容と厳密に一致する（resume 未対応の葉は再保存で agent 情報が落ちるため、以後のディスク内容とは乖離しうる）。

並びは、起源 workspace（配列先頭・起動時の最初の workspace）を常に最上位に固定し、残りを最終使用時刻（タブ選択を含むアクティブ化のたびに更新・永続する → [persistence](persistence.md)）の新しい順（MRU）で並べる。同時刻・未設定（旧データ）同士は元配列順で安定化。並べ替えは表示順のみで配列自体は不変（切替は元オフセットで駆動）。

このセッションで一度も表示していない休眠 workspace（永続復元のみ・未だ切替ていない＝surface 未起動）は行全体を減光する（選択時は減光解除）。減光は MRU 位置とは独立した別軸で、末尾固定はしない。休眠状態は永続しない（切替で起こすと解除、次回起動でまた休眠から）。全 agent が idle でも未起動でなければ減光しない。

## 専用作成フォーム（ソース切替: 既存フォルダ / git clone）

切替パレット一覧の末尾に常設の作成導線行「＋ 新規ワークスペース — パスから作成」（accent 文字・破線罫線・右端 `⌘N` バッジ。絞り込み中も末尾に残す）を出す。この行の Enter／クリック、または端末フォーカス時の `⌘N` で専用の作成オーバーレイへ入る（chrome ショートカットは端末 SurfaceView が先取りするため、切替パレットの入力欄フォーカス中は `⌘N` が発火しない＝行のバッジは global アクションの表示）。カードは切替パレットと同位置・同寸法で「その場が差し替わる」。ヘッダ（「新規ワークスペース」＋「esc 戻る」ピル）／ソース切替タブ／本文（ソース別）／フッター（案内＋「キャンセル」「作成して開く」）。

作成フォーム表示中は常にいずれかの入力欄が focus される（表示直後・ソース切替直後・補完確定後・relink 後）ため esc・↑↓・⇥ が常に効く。overlay 間の遷移（作成↔切替パレット↔端末）でも focus は遷移先へ確実に移り、去りゆくカードに奪われない。

**ソース切替タブ**: ピル 2 つ（「既存フォルダ」／「git clone」）。選択中は accent、非選択は muted。切替で本文が差し替わる。切替時は各ソースの入力（folder のパス、clone の URL/clone 先）を保持し、名前だけ追従（linked）へ戻す——導出元が変わるため。名前の追従／リンク解除／再リンクは folder/clone で同じ 1 経路を共用する。

### 既存フォルダソース

- **パス欄（主役・mono 入力）**: 初期値はアクティブペインの cwd（`~` 短縮）、無ければ `~`。入力の末尾セグメントで前方一致した実在の子ディレクトリを、パス欄直下のドロップダウンに出す。各行はフォルダ名＋`~` 短縮フルパス＋`.git` があれば右端に green の「git」タグ。↑↓ 移動・⇥/↵ 確定（パスをそのフォルダへ差し替え・名前を再リンク・focus はパス欄に残す）・esc で候補ドロップダウンだけを閉じる（パスは保持・画面は戻さない）。ドロップダウンが閉じている（候補なし）ときのパス欄は ↵＝作成・esc＝前の画面へ戻る。隠しフォルダ（`.` 始まり）は末尾セグメントが `.` 始まりのときだけ含める。補完列挙は純関数を背景 queue で呼び main へ hop（stale は破棄）。
- **名前欄がパスに追従**: パス末尾セグメントから名前を導出して表示し、「⌁ パスに追従中」を出す。名前を手入力するとリンク解除し「リンク解除中 — 再リンク」ボタンを出す（押すと追従へ戻る）。
- **作成の契約**: 「作成して開く」/↵ で、入力パスを rootPath・名前を name とする workspace を作成しアクティブ化して開く（`~` は store 展開・rootPath でシェル 1 枚を起こす）。フッター案内は実在時「作成すると {名前}（{パス}）が開きます」。
- **不在パスの契約（folder 限定）**: 入力パスが実在するディレクトリでないときは「作成して開く」を無効化し、フッター案内を「フォルダが存在しません」に差し替える（フォルダ新規作成は対象外）。

### git clone ソース

- **リポジトリ URL 欄（mono 入力）**: 初期値は空・プレースホルダに例を薄字表示（IME 変換中は非表示＝chrome 入力欄の共通挙動 → [dispatch-palette](dispatch-palette.md)）。URL は**正規化せず素通し**する（git が `https://` / `ssh://` / scp-like `git@host:owner/repo.git` を native 解釈するため）。`GIT_TERMINAL_PROMPT=0` を与えて資格情報プロンプトでハングさせず stderr へ倒す。
- **clone 先欄（mono 入力・親ディレクトリ）**: 初期値は folder のパスと同じ導出。補完ドロップダウンは無し。欄の末尾に `/{名前}`（編集可）を並べ、実際の clone 先 `clone先/名前` を示す。下に注記「フォルダは作成時に作られます」。
- **名前の追従/手入力**: 名前は URL 末尾セグメントから `.git` を除いて自動導出（`repo.git`→`repo`、`git@github.com:you/repo.git`→`repo`、空なら `workspace`）。手入力でリンク解除（folder と同じ・追従の導出元は URL）。
- **作成可否の契約**: 「作成して開く」/↵ は URL 非空（trim 後）＋clone 先の親が実在ディレクトリ＋名前非空で有効。clone 先の既存衝突は事前チェックせず、実行して git の stderr に委ねる。
- **clone 実行（待機・キャンセル不可）**: headless で `git clone <url> <dest>` を実行する（git 知識は Git 層に閉じ、フォームは clone runner を closure で受ける）。実行中は本文をスピナーの待機表示へ差し替え、両ボタンを無効化し esc / Scrim タップも無視する——**中断ハンドルが無いため完了まで待つしかない**（二重実行は submit 側の guard で防止）。成功すると clone 先を rootPath として workspace を作りその場で開く。失敗は git の stderr をフォーム内に inline 表示（danger）し、フォームを操作可能に戻して再実行できる。
- **戻り／非破壊**: 「キャンセル」/`esc`/Scrim タップで切替パレットへ戻る（clone 待機中を除く）。既存のクイック作成（一致しない名前＋Enter）はそのまま並置で残す。

## 設定上書き（workspace 毎プロファイル）

各 workspace は**全設定**の上書きを 1 つの均一な設定層で持ち、永続する（→ [persistence](persistence.md)）。値の担体がスコープ非依存の単一型なので、gui.conf 経由の設定も、gui.conf を経由せず chrome へ直配信する設定（エージェントアイコン・タブタイトルフォント → [chrome](chrome.md)）も、起動系が読む設定（デフォルトエージェント）も、UI gate（開発中の機能 → [editor-pane](editor-pane.md)）も一律に上書きできる。空層は「上書き無し」へ畳む。

解決は global 層に当該 workspace の上書き層を重ねた**実効設定**（項目ごとに「上書き ?? global ?? 既定」。エージェントアイコンのマップだけは非 nil ならマップ全体を差し替える＝per-key マージしない）。反映は集約点 `applyActiveWorkspaceConfig()`（外観同期＋gui.conf 再生成＋状態アイコン更新＋右バー gate 再評価）が担い、アクティブ化（workspace 切替・起動復元・空 workspace アクティブ化）・初回起動・workspace 作成・設定パレット適用で呼ぶ——**画面に載るのは常にアクティブ 1 workspace のみ**なので、gui.conf 再生成＋全 surface 一律 reload で常に正しい。上書きの編集は設定パレットのスコープトグル（[settings-palette](settings-palette.md)）。上書きの無い workspace は global で動く。
