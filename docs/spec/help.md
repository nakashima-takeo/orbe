---
title: ヘルプオーバーレイ（⌘H）
description: ⌘H でトグル開閉するショートカットチートシート — トップビュー（厳選＋状態凡例）・検索×カテゴリ×キー絞り込みの一覧・物理キーボード可視化
updated: 2026-07-24
---

`⌘H` でヘルプオーバーレイ（ショートカットチートシート）がトグル開閉する。⌘H は chrome キーとして macOS Hide より先に消費され、メニューの「Orbe を隠す」は項目のみ残りショートカット表記を持たない（クリックでは従来どおり隠れる。⌘⌥H の「ほかを隠す」は従来どおり）。0 タブでも開く。他のオーバーレイ表示中は不活性（他 chrome キーと同じ規律）。esc・暗幕クリックでも閉じ、閉じたら端末へフォーカスが戻る。

ヘルプ表示中は ⌘ 併用ショートカットの実動作をすべて抑止する（メニューの keyEquivalent 含む。⌘Q で終了しない）——ヘルプはショートカットを試し押しして確かめる場で、押下はキーボード可視化の点灯と該当行ハイライトにのみ使われる。表示中に実動作するのは ⌘H（トグルで閉じる）と esc のみ。閉じた後は全ショートカットが従来どおり。

表示は最暗の暗幕＋中央のモーダルカード（幅・高さは上限つきで窓に収まる）。窓がさらに小さいときは二段階で縮む——内部リフローが成立する床まではリフロー（コンテンツはスクロール）し、床を下回るとカード全体を等倍縮小する（キーボード可視化が固定アスペクトの図であるため、部分的な折返しでなく全体スケールで構成を保つ）。掲載内容は静的カタログが持ち、実在する Orbe のキーバインドのみを載せる（カタログの内部整合はテストが機械保証。実バインドとの突合は棚卸しが正）。全 UI 文言は現在言語に追従する（[localization](localization.md)）。

**検索欄**: 開いた瞬間からフォーカスされ、そのままタイプで検索できる。esc は状態に依らず常に閉じる（絞り込みの解除はチップの × / 使用キーの再クリック / 文字削除で行う）。

**トップビュー（基本操作）**: 初期表示。厳選ショートカット 3 グループ（全般 / ワークスペースとタブ / エージェント）の 2 カラムグリッド＋エージェントステータス凡例（working / waiting / done / idle。グリフは実アプリと同一の状態グリフで、モーション・Reduce Motion 時の静止も同じ）。

**一覧ビュー**: 検索（ラベル・キー表記への部分一致）× カテゴリ（サイドバー選択）× キー絞り込みの AND。ヘッダに件数、絞り込み中はキーのチップ（タップで解除）。「基本操作」選択中でも検索・キー絞り込みがあると一覧へ自動遷移し、クリアでトップへ自動復帰する。空になったカテゴリは見出しごと消える。

**キーボード可視化**: モーダル下部に物理 6 段配列。点灯＝行ホバーの該当キー ∪ 実押下キー ∪ キー絞り込み選択。実押下は表示中のみのローカルイベント監視で拾い（⌘ 併用の keyDown は上記のとおり消費して点灯専用。keyUp・修飾キー単独・非修飾のタイプは素通しで検索入力と共存）、修飾キーは左右を区別する。⌘ を離すと非修飾キーの残留点灯を全消灯する（macOS は ⌘ 押下中の他キー keyUp を配らないため）。ウィンドウが key を失っても全消灯。ショートカットに登場するキーはクリックで絞り込みをトグルでき、登場しないキーは暗く表示されクリックに反応しない。

**実押下の行ハイライト**: 押下中のキー集合（左右修飾は同一視）がいずれかの行の組み合わせと完全一致すると、その行がハイライトされる（同じ組み合わせの行は全行）。該当行が現在のビューに無ければ「すべて」一覧へ自動遷移し（検索・キー絞り込みが該当行を隠す場合のみクリア）、画面外なら自動スクロールで見せる。キーを離すとハイライトは消えるがビューは戻らない。⇧併用で記号が変わるキー（⌘⇧] 等）も元のキーに解決して一致する。
