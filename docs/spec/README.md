# spec — 現状仕様

Orbe の「今どうなっているか」だけを機能単位で持つ。

## 規約

- **現状のみ。** 理由・履歴・経緯は書かない。「なぜ今こうか」は当該変更の PR 本文が持つ。
- **実装詳細を書かない。** 書くのは変わるとき人間の意思決定が要るもの——挙動・契約・境界。コードを開けば分かる定数・フィールド名・レイアウト数値は書かない。ただし**その値でなければならない理由（本質的なロジック）は一文残す**。
- 全ページに frontmatter（`title` / `description` 1行要約 / `updated: YYYY-MM-DD`）を持たせる。
- 参照は相対リンク。1枚に育ちすぎたら割る。
- **`Sources/` を変更したセッションは、終了前に該当 spec を現状に合わせて更新する。**
