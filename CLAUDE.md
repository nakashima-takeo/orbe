# Orbe

- 現状仕様は `docs/spec/` が持つ（規約は `docs/spec/README.md`）。**`Sources/` を変更したセッションは、終了前に該当 spec を現状に合わせて更新する**（書くのは現状のみ。理由は PR 本文に）。
- アイデア・要望・不具合・積んである作業は GitHub Issue が持つ。

## 原則

- **開発ワークフローを本質的に変える**のが Orbe の目的。最適化は極限まで行う。
- **手順の簡略化**が本質的価値。価値は、今まで気にしていなかった手間をなくすことにある（例: ワンアクションで「エージェントが起動した状態のタブ」が開く）。上とは独立した別の価値。
- **他実装の挙動再現を完了条件にしない。** 仕様は Orbe の設計として一から立てる。
- **オーバーレイは覚える近道を最小に。** 1つのショートカットで開き、基本操作は Enter、詳細は右矢印でサブメニューに潜る。
- **画面確認は本物の中身で preview（静止）→ flow（アクションで現れる状態）の順。Peekaboo（外部 GUI 自動化）は最終手段。** preview/flow は in-process・決定論・権限不要。mechanics は visual-check スキルが持つ。
